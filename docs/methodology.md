# Methodology

## Scope

This repository documents only the public and academic evidence needed to understand the article's methodological reasoning.

## Evidence sources

Evidence is derived from the article and represented as public summaries, traceability tables, reference diagrams and non-operational IaC examples.

## Methodological artifacts

| Artifact | Purpose | Repository location |
| --- | --- | --- |
| TOGAF domain diagnosis | Organize the AS-IS architecture by domain. | `evidences/togaf-domains/` |
| ICA | Identify architectural criticality by process. | `evidences/traceability-matrix.md` |
| BCM | Link capabilities with supporting domains. | `architecture/as-is-summary.md` |
| IMSA | Select the prioritized architectural gap. | `evidences/traceability-matrix.md` |
| AS-IS/TO-BE indicators | Compare current and target states. | `architecture/to-be-reference-architecture.md` |
| Risk matrix | Link risks to controls. | `risk/risk-register.md` |

## Reproducibility limits

The repository is reproducible at the methodological level. It is not intended to reproduce real operational infrastructure, internal software or production data.
